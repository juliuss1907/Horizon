Part 4 of the Building EDIFlow series demonstrates Infrastructure Layer implementation in TypeScript, showing how abstract interfaces (IMessageParser, IMessageStructureRepository) are concretely implemented as EdifactMessageParser and FileBasedMessageStructureRepository. The architecture splits across three packages (@ediflow/edifact, @ediflow/x12, @ediflow/infrastructure-shared) to handle four EDI standards (EDIFACT, X12, HIPAA, EANCOM) with zero shared code between standard-specific packages. Clean Architecture implementation in TypeScript is widely discussed but often poorly understood in practice. This tutorial provides concrete guidance on separating standard-specific implementations from shared infrastructure, which is directly applicable to developers building complex enterprise applications with multiple data standards. The three-package architecture demonstrates how to avoid God-packages while maintaining clear dependency boundaries. The architecture prevents circular dependencies by having every infrastructure package depend only on @ediflow/core (for interfaces), never on each other. EDIFACT uses delimiters '+:.''' while X12 uses '*~>' with completely different envelope structures (UNB/UNZ vs ISA/GS/ST). The infrastructure-shared package was created primarily for the CLI tool, which needs to load message definitions for all standards from a single entry point using FileBasedMessageStructureRepository.

rss · Dev.to · May 7, 03:44

**Background**: Clean Architecture is a software design philosophy that organizes code into layers (Domain, Application, Infrastructure, Interface Adapters) with strict dependency rules—inner layers never depend on outer layers. EDIFACT (UN/Electronic Data Interchange For Administration, Commerce and Transport) is an international EDI standard developed by the UN, widely used in global supply chain and logistics. X12 is the ANSI-standard EDI format predominant in North American healthcare and retail sectors. The dependency graph shows @ediflow/infrastructure-shared depending on @ediflow/core, which in turn @ediflow/cli depends on for wiring all packages together.

## [🚨 Origin Protocol exploit 🚨](https://x.com/originprotocol/status/2052125698983420122?s=12&t=2q9OLMlaobNlHLewOXT4uA) ⭐️ 4.0/10

A flash loan attack on Origin Protocol's Morpho OETH pool exploited a permissionless rebase() function to claim yield. The attacker borrowed approximately 7,557 OETH, called rebase() to harvest rewards, swapped the assets on Curve, and repaid the loan, netting approximately 0.148 OETH (~$350). This incident highlights a vulnerability in how permissionless rebase() functions interact with yield-bearing tokens in DeFi composability. Even with 12+ audits and 4+ years of battle-tested code, the OETH protocol's integration with Morpho created an exploitable attack vector that could inspire similar attacks on other protocols. The attack leveraged 7,557 OETH as flash loan capital and exploited Curve liquidity for asset swaps. The modest profit of ~$350 (0.148 OETH) suggests this may have been a proof-of-concept attack or white-hat demonstration rather than a large-scale exploit. The incident raises questions about the safety of permissionless rebase() calls in yield-bearing token architectures.

telegram · ahboyashreads · May 7, 03:57

**Background**: OETH (Origin Ether) is a liquid staking token that offers higher staking rewards with deep liquidity and DeFi composability. The protocol shares its codebase with OUSD and has undergone 12+ audits. Morpho is a DeFi protocol that optimizes lending markets by matching lenders and borrowers directly. Flash loans allow users to borrow assets without collateral within a single transaction, making them popular for arbitrage and exploit attacks.
Rebasing tokens adjust their supply algorithmically to maintain price stability, which can create unintended yield distribution when combined with other DeFi protocols. The rebase() function in OETH appears to have allowed anyone to trigger yield claims on behalf of depositors, creating a vulnerability exploitable via flash loans.
